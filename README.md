# FinanceMobile 💰

Кроссплатформенное приложение для учета личных финансов (доходы, расходы, депозиты и счета). Построено на фреймворке Avalonia UI с использованием паттерна MVVM.

## 🛠 Стек технологий

- **Язык**: C# (.NET)
- **UI-фреймворк**: Avalonia UI
- **База данных**: SQLite (через прослойку `DatabaseService`)
- **Архитектура**: MVVM (Model-View-ViewModel), Singleton для основной бизнес-логики

---

## 🏗 Архитектура приложения

Ядром бизнес-логики является класс **`Budget`**. Он реализован как Singleton и служит единой точкой входа для всех операций с данными.

### Иерархия данных
1. **`Budget`** (Singleton) – управляет всем состоянием. При инициализации автоматически загружает данные из БД через метод `LoadDataFromDB()`.
2. **`Section`** – разделы финансов (по умолчанию: `"Incomes"` и `"Expenses"`).
3. **`Category`** – категории внутри раздела (например, "Еда", "Зарплата").
4. **`SortedOperationList`** – специализированная коллекция операций внутри категории. Реализована по принципу связного списка (`LinkedList` с узлами `ListNode`), **всегда отсортирована по дате** (от ранней к поздней). Поддерживает периодические (повторяющиеся) операции.
5. **`Operation`** – единичная финансовая операция (дата, сумма, ID счета).
6. **`Deposite`** и **Счета (Accounts)** – хранятся в отдельных словарях внутри `Budget`.

### Взаимодействие с базой данных
Класс `Budget` **полностью инкапсулирует** работу с базой данных. Он самостоятельно вызывает методы `DatabaseService` для сохранения или загрузки данных. 

> ⚠️ **Золотое правило для разработчиков:** ViewModel **не должны** обращаться к `DatabaseService` напрямую. Все изменения состояния должны происходить исключительно через публичные методы класса `Budget`.

## 🚀 Как добавить новый экран (UI)

В приложении используется навигация через ViewModel. Чтобы добавить новый экран, выполните 3 шага:

### Шаг 1: Создайте ViewModel и View
Создайте класс ViewModel (наследник `ViewModelBase`) и соответствующий ему `.axaml`-файл (View).

### Шаг 2: Зарегистрируйте DataTemplate
Откройте файл `App.axaml` и добавьте связку вашей ViewModel и View в секцию `<Application.DataTemplates>`:

    <Application.DataTemplates>
        <!-- Пример существующих связок -->
        <DataTemplate DataType="vm:BudgetViewModel">
            <v:BudgetView />
        </DataTemplate>
        
        <!-- ДОБАВЬТЕ СЮДА ВАШУ СВЯЗКУ -->
        <DataTemplate DataType="vm:YourNewViewModel">
            <v:YourNewView />
        </DataTemplate>
    </Application.DataTemplates>

*(Не забудьте добавить соответствующие пространства имен `xmlns:vm` и `xmlns:v` в корневой тег `<Application>` файла `App.axaml`, если их там еще нет).*

### Шаг 3: Вызовите навигацию
Для перехода на новый экран используйте статический метод `App.NavigateTo`. Обычно это делается внутри команды (Command) вашей текущей ViewModel.

**Пример использования во ViewModel:**
```csharp
using System.Windows.Input;
using CommunityToolkit.Mvvm.Input; // Или ваша библиотека для команд

public class SomeViewModel : ViewModelBase
{
    public ICommand GoToAddEntryCommand { get; }

    public SomeViewModel()
    {
        // При вызове команды создаем новую ViewModel целевого экрана и передаем её в навигатор
        GoToAddEntryCommand = new RelayCommand(() => 
        {
            App.NavigateTo(new AddEntryViewModel());
        });
    }
}
```

**Пример с передачей параметров:**
```csharp
public ICommand OpenDetailsCommand { get; }

public SomeViewModel()
{
    // Если целевому экрану нужны данные, передайте их через конструктор
    OpenDetailsCommand = new RelayCommand<int>((categoryId) => 
    {
        App.NavigateTo(new CategoryDetailsViewModel(categoryId));
    });
}
```

## 📚 Основные методы класса `Budget` (API)

Используйте эти методы из ваших ViewModel для взаимодействия с данными. 

> 💡 **Совет следующей команде:** При добавлении новых методов в класс `Budget` или `Section`, пожалуйста, документируйте их с помощью стандартных XML-комментариев C# (`///`), как показано в примере ниже. Это позволит IDE автоматически показывать подсказки и упростит работу новым разработчикам.

**Пример правильного документирования метода:**
```csharp
/// <summary>
/// Добавляет новую финансовую операцию в указанную категорию.
/// </summary>
/// <param name="sectionName">Имя раздела ("Incomes" или "Expenses").</param>
/// <param name="categoryName">Имя категории.</param>
/// <param name="date">Дата операции.</param>
/// <param name="accountID">Идентификатор счета.</param>
/// <param name="value">Сумма операции.</param>
/// <param name="isPlanned">Является ли операция плановой (true) или фактической (false).</param>
/// <param name="periodInDays">Интервал повторения в днях (-1 для разовой операции).</param>
/// <param name="endDate">Дата окончания периодической операции (опционально).</param>
public void AddOperation(string sectionName, string categoryName, DateTime date, 
    string accountID, double value, bool isPlanned = false, 
    int periodInDays = -1, DateTime? endDate = null)
{
    // ...
}
```

**Краткий справочник по основным методам:**

| Метод | Описание |
| :--- | :--- |
| `AddAccount(string name, double initialBalance)` | Добавляет новый счет с начальным балансом. |
| `AddDeposite(string name)` | Добавляет новый депозит. |
| `AddCategory(string sectionName, string categoryName)` | Создает новую категорию в разделе "Incomes" или "Expenses". |
| `AddOperation(...)` | Добавляет операцию. Поддерживает параметры `isPlanned` (план/факт) и `periodInDays` (для периодических операций). |
| `GetOperations(...)` | Возвращает `IEnumerable<Operation>` отфильтрованных операций (можно указать `dateStart`, `dateEnd`, `isPlanned`). |
| `GetSectionSum(...)`, `GetCategorySum(...)` | Возвращает агрегированные суммы (`double`) по разделу или категории за указанный период. |
| `GetIncomeExpenseDifference(...)` | Возвращает разницу между доходами и расходами (текущий баланс). |
| `GetAccountNames()`, `GetDepositeNames()` | Возвращают перечисления (`IEnumerable<string>`) имён счетов и депозитов. |

## ⚠️ Примечания для будущих разработчиков (Technical Debt)

При продолжении разработки обратите внимание на следующие места в коде, требующие доработки или рефакторинга. Для каждого пункта указана рекомендация по исправлению:

### 1. Хардкод типа счета
- **Проблема:** В методе `Budget.AddAccount` тип счета жестко задан как `AccountType.Card`.
- **Как исправить:** Вынести параметр `AccountType` (или строковое представление) в аргументы метода `AddAccount` и добавить выбор типа в UI.
- **Файл:** `DataStructs/Budget.cs`, метод `AddAccount`.

### 2. Периодические операции и счета
- **Проблема:** В методе `Budget.AddOperation` при сохранении `PeriodicOperation` поле `AccountId` оставляется пустым (`""`).
- **Как исправить:** Передать `accountID` в объект `PeriodicOperation` и обеспечить корректную привязку будущих генерируемых операций к этому счету.
- **Файл:** `DataStructs/Budget.cs`, метод `AddOperation`.

### 3. Цвета категорий
- **Проблема:** В методе `AddCategory` поле `Color` сохраняется как пустая строка `""`.
- **Как исправить:** Добавить параметр `color` (например, HEX-строку) в метод `AddCategory` и соответствующий элемент управления в UI создания категории.
- **Файл:** `DataStructs/Budget.cs`, метод `AddCategory`.

### 4. Дублирование сущностей (Singleton)
- **Проблема:** В комментариях упоминается `App.AppBudget`, но в коде используется `Budget.BudgetInstance`.
- **Как исправить:** Унифицировать доступ к единственному экземпляру. Рекомендуется использовать встроенный механизм DI (Dependency Injection) в Avalonia или оставить один статический геттер в классе `App`, который возвращает `Budget.BudgetInstance`.
- **Файлы:** `App.axaml.cs`, `DataStructs/Budget.cs`.

### 5. Логика сохранения периодических операций
- **Проблема:** В `AddOperation` присутствует цикл для сохранения периодических операций, помеченный комментарием *"для десктопа сохраняю по их правилам, сам игнорирую эти записи"*. Это ведет к дублированию записей в БД.
- **Как исправить:** Пересмотреть стратегию хранения. Хранить только шаблон `PeriodicOperation` в отдельной таблице, а фактические операции генерировать "на лету" при запросе `GetOperations` или в фоновом задании при наступлении даты, а не создавать сотни записей заранее.
- **Файл:** `DataStructs/Budget.cs`, метод `AddOperation`.

### 6. Отсутствие валидации входных данных
- **Проблема:** Методы `Budget` не проверяют корректность входных данных (например, отрицательные суммы, пустые имена).
- **Как исправить:** Добавить валидацию в начале публичных методов и выбрасывать осмысленные исключения (`ArgumentException`, `InvalidOperationException`).

## 🏁 Быстрый старт

1. Убедитесь, что у вас установлен **.NET SDK** (версия 8.0 или выше).
2. Клонируйте репозиторий:
   ```bash
   git clone <url-репозитория>
   cd FinanceMobile
   ```
3. Восстановите пакеты:
   ```bash
   dotnet restore
   ```
4. Запустите приложение:
   ```bash
   dotnet run
   ```
